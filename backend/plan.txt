1.1 — Fix the insecure JWT default in docker-compose.yml

The docker-compose.yml has JWT_SECRET:-production_ready_secret_key_123 as a fallback. If .env is missing, the app starts with a known-public secret. Fix it to crash instead.



Command for agent

In docker-compose.yml, find the line:

Replace it with:

This uses bash :? syntax which makes Docker Compose exit with an error if the variable is unset, preventing the app from starting with a weak secret.

1.3 — Add rate limiting to the /api/auth/login endpoint

The auth router already has a check_rate_limit() helper function, but it is not wired to the login endpoint correctly. The current implementation has a logic bug: it uses SET NX EX which means the TTL is only set on first creation — if you hit the limit and wait, the counter never resets. Fix the sliding window logic.



Command for agent

In backend/app/api/routers/auth.py, replace the check_rate_limit function with:



async def check_rate_limit(key: str, limit: int, window: int):

    pipe = redis_client.pipeline()

    await pipe.incr(key)

    await pipe.expire(key, window)

    results = await pipe.execute()

    count = results[0]

    if count > limit:

        raise HTTPException(

            status_code=429,

            detail=f'Too many attempts. Try again in {window // 60} minutes.'

        )



This uses a pipeline so both INCR and EXPIRE are atomic. The expire is set on every call, making it a true sliding window. The current code only sets the TTL on first creation (NX), so a burst that hits the limit can never reset.

Phase 2: Campaign System — Fix WhatsApp Template Compliance  ● Critical for production campaigns



Your campaign system currently sends freeform messages to customers. WhatsApp's Business API (since July 2025) requires all business-initiated messages outside the 24-hour customer service window to use pre-approved Message Templates (HSMs). Without this, your bulk campaigns will fail with a 131026 or 132000 error from Meta in production.

5.2 — Create a knowledge base ingestion service

Command for agent

Create backend/app/services/knowledge_service.py with:



import openai, textwrap

from app.models.knowledge_chunk import KnowledgeChunk

from app.core.config import settings



async def ingest_text(db, business_id, text: str, source: str):

    # Delete old chunks for this source

    await db.execute(delete(KnowledgeChunk).where(

        KnowledgeChunk.business_id == business_id,

        KnowledgeChunk.source == source

    ))

    # Chunk into ~400-word pieces with 50-word overlap

    words = text.split()

    chunks = [' '.join(words[i:i+400]) for i in range(0, len(words), 350)]

    client = openai.AsyncOpenAI(api_key=settings.OPENAI_API_KEY)

    for chunk in chunks:

        resp = await client.embeddings.create(input=chunk, model='text-embedding-ada-002')

        embedding = resp.data[0].embedding

        db.add(KnowledgeChunk(business_id=business_id, source=source, content=chunk, embedding=embedding))

    await db.commit()



async def search_knowledge(db, business_id, query: str, top_k=5) -> str:

    client = openai.AsyncOpenAI(api_key=settings.OPENAI_API_KEY)

    resp = await client.embeddings.create(input=query, model='text-embedding-ada-002')

    qvec = resp.data[0].embedding

    results = await db.execute(

        select(KnowledgeChunk.content)

        .where(KnowledgeChunk.business_id == business_id)

        .order_by(KnowledgeChunk.embedding.cosine_distance(qvec))

        .limit(top_k)

    )

    chunks = results.scalars().all()

    return '\n\n'.join(chunks) if chunks else ''
